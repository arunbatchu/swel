# Concept Taxonomy for Software Engineering Leaders

This document defines the categorical taxonomy for organizing the 200 concepts in the learning graph.

## Taxonomy Categories

| Category Name | TaxonomyID | Description |
|--------------|------------|-------------|
| Foundation | FOUND | Core leadership concepts, foundational principles, and entry-point topics |
| Cloud Infrastructure | CLOUD | Cloud computing, containers, Kubernetes, Infrastructure as Code, serverless |
| AI and Machine Learning | AIML | Large Language Models, MLOps, AI-assisted development, data streaming |
| Developer Experience | DEVEX | Platform engineering, observability, tooling, feature flags, APIs |
| Team Leadership | TEAM | Team topologies, hiring, interviewing, career development, retention |
| Team Culture | CULT | Psychological safety, remote work, distributed teams, collaboration |
| Engineering Practices | PRACT | Code quality, testing, CI/CD, technical debt, architecture decisions |
| Operations | OPS | Site Reliability Engineering, incident management, on-call, postmortems |
| Metrics | METR | DORA metrics, SPACE framework, engineering measurement, business outcomes |
| Culture Patterns | CPAT | Engineering culture, anti-patterns, innovation, craftsmanship |
| Strategy | STRAT | Technology strategy, stakeholder management, build vs buy decisions |
| Organization | ORG | Conway's Law, organizational design, team scaling, specialization |
| Emerging Tech | EMERG | AI agents, quantum computing, Web3, future technologies |

## Category Descriptions

### FOUND - Foundation

Core concepts that establish the baseline understanding for engineering leadership. These are typically prerequisites for most other topics and include fundamental definitions and principles.

**Example concepts**: Software Engineering Leader, Engineering Organization, Technical Leadership, People Management, Engineering Strategy

### CLOUD - Cloud Infrastructure

Technologies and practices related to cloud computing, containerization, and infrastructure management. Covers major cloud providers, orchestration tools, and deployment patterns.

**Example concepts**: Kubernetes, Docker, AWS, Azure, GCP, Terraform, Serverless Architecture, Edge Computing

### AIML - AI and Machine Learning

Artificial intelligence and machine learning concepts relevant to engineering leaders. Includes LLMs, AI-assisted development tools, MLOps practices, and data technologies.

**Example concepts**: Large Language Models, GitHub Copilot, MLOps, Model Versioning, Vector Databases, RAG Architecture

### DEVEX - Developer Experience

Tools and practices that improve developer productivity and experience. Covers platform engineering, observability, feature management, and API design.

**Example concepts**: Platform Engineering, Internal Developer Platform, Observability, Feature Flags, API Management

### TEAM - Team Leadership

Concepts related to building, hiring, and developing engineering teams. Includes team structures, interviewing, career development, and talent management.

**Example concepts**: Team Topologies, Stream-Aligned Teams, Structured Interviewing, Career Ladders, Skills Gap Analysis

### CULT - Team Culture

Interpersonal and cultural aspects of team dynamics. Covers psychological safety, remote work practices, and distributed team collaboration.

**Example concepts**: Psychological Safety, Trust Building, Remote Team Leadership, Asynchronous Communication, Documentation Culture

### PRACT - Engineering Practices

Technical practices and processes for software development excellence. Includes code quality, testing strategies, continuous integration, and architecture.

**Example concepts**: Code Review Process, Unit Testing, Continuous Integration, Technical Debt, Architecture Decision Records

### OPS - Operations

Operational excellence and reliability engineering concepts. Covers SRE principles, incident management, on-call practices, and system reliability.

**Example concepts**: Site Reliability Engineering, Service Level Objectives, Incident Management, Blameless Postmortems, On-Call Practices

### METR - Metrics

Engineering metrics and measurement frameworks. Includes DORA metrics, developer productivity measurement, and outcome-focused metrics.

**Example concepts**: DORA Metrics, SPACE Framework, Deployment Frequency, Lead Time for Changes, Cycle Time

### CPAT - Culture Patterns

Engineering culture, both positive patterns and anti-patterns to avoid. Covers organizational culture, innovation practices, and sustainability.

**Example concepts**: Engineering Culture, Innovation Mindset, Blame Culture, Hero Culture, Team Health Checks

### STRAT - Strategy

Strategic leadership concepts for engineering executives. Covers technology strategy, stakeholder management, and executive communication.

**Example concepts**: Technology Strategy, Build vs Buy, Stakeholder Management, Executive Communication, Technical Due Diligence

### ORG - Organization

Organizational structure and design concepts. Covers Conway's Law, scaling teams, and organizational patterns.

**Example concepts**: Conway's Law, Organizational Design, Team Scaling, Specialization Balance

### EMERG - Emerging Tech

Emerging technologies and future-oriented concepts. Covers new technology trends and preparing organizations for change.

**Example concepts**: AI Agents, Quantum Computing, Web3 Technologies, Continuous Learning, T-Shaped Engineers

## Distribution Target

- Target: 12-13 categories
- Goal: Even distribution (~15-20 concepts per category)
- Maximum: No category should exceed 30% of total concepts (60 concepts)
